## 最低限おさえておくこと

付録を読んだあと，以下の概念が理解できているか確認しましょう．

| 概念 | 一言説明 |
|------|---------|
| クラス | データと関数をひとまとめにした設計図 |
| オブジェクト | クラスから生成した実体 |
| メンバ変数 | クラスが持つデータ（末尾 `_` が慣習）|
| コンストラクタ | オブジェクト生成時に自動で呼ばれる初期化関数 |
| デストラクタ | オブジェクト破棄時に自動で呼ばれる後片付け関数 |
| メンバ関数 | クラスが持つ処理 |
| public / private | 外から使えるか（public）・クラス内部だけか（private）|
| 継承 | 既存クラスを拡張して新しいクラスを作る |
| `this` | 自分自身のオブジェクトへのポインタ |

## ROS2 でクラスが重要な理由

ROS1 では `main()` 関数の中に直接 `ros::NodeHandle nh` を作ってノードを書くことが多くありましたが，ROS2 では **`rclcpp::Node` を継承したクラス** として書くのが標準スタイルです．

理由は 3 つあります：

1. `rclcpp::Node` は多くの機能（Publisher・Subscriber・Timer・パラメータ等）を提供するクラスで，**継承することで `this->create_publisher()` などが使えるようになる**
2. **コールバック関数** からメンバ変数へのアクセスが自然にできる
3. `std::make_shared<MyNode>()` を使うことでメモリ管理がシンプルになる
